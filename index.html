Перелік основних команд:

  • git config --global user.name blabla-зберігає інформацію для користувача
  • git config --global user.email blabal@bla-зберігає інформацію для електронної почти
  • git init - створити порожнє сховище Git або повторно ініціалізувати існуюче
  • git status - Показує статус робочого дерева
  • git add - Додати вміст файлу до індексу
  • git commit - записує зміни до репозиторію
  • git log - Показати журнали комітів
  • git checkout - перемикання гілок або відновлення робочих файлів дерева
  • git branch - список, створення або видалення гілок
  • git push - Оновлення віддалених посилань разом із пов’язаними об’єктами
  • git pull - Отримання та інтеграція з іншим репозиторієм або локальною гілкою
  • git merge - об'єднайте дві або більше історій розробки разом
  • git clone - клонувати репозиторій у новий каталог

Файл .gitignore - визначає навмисно невідстежувані файли, які потрібно ігнорувати

Advanced

  • git revert - повертає деякі існуючі коміти
  • git reset - Скидання поточного HEAD до вказаного стану
  • git rebase - повторне застосування комітів поверх іншої базової підказки
  • git cherry - pick-Застосувати зміни, внесені деякими існуючими комітами
  • git stash - Зберігає зміни в брудному робочому каталозі
